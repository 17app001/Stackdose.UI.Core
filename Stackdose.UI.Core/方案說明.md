# Stackdose.UI.Core 方案說明

## 1. 專案定位

`Stackdose.UI.Core` 是工業場景的 WPF 核心 UI 函式庫，目標是把「可重用的共通能力」集中在同一層，讓上層應用（例如模板/機台專案）可以快速組裝介面與流程，而不必重複造輪子。

核心關注：

- PLC 狀態與設備互動 UI
- 使用者權限與登入會話
- 合規/稽核日誌（含批次寫入）
- 可維護的主題與設計 Token
- Shell 導航共用契約（支援多 App 複用）

## 2. 技術基線

- Target Framework: `net8.0-windows`
- UI: `WPF` (`UseWPF=true`)
- 語言設定: Nullable 開啟、Implicit Usings 開啟
- 資料存取: `SQLite` + `Dapper`
- AD 驗證能力: `System.DirectoryServices.AccountManagement`
- 平台目標: `x64`

關鍵相依（需與此專案同層存在，否則完整編譯可能失敗）：

- `..\..\Stackdose.Platform\Stackdose.Abstractions\Stackdose.Abstractions.csproj`
- `..\..\Stackdose.Platform\Stackdose.Core\Stackdose.Core.csproj`
- `..\..\Stackdose.Platform\Stackdose.Hardware\Stackdose.Hardware.csproj`
- `..\..\Stackdose.Platform\Stackdose.PrintHead\Stackdose.PrintHead.csproj`
- `FeiyangWrapper.dll`（Debug/Release 路徑條件載入）

## 3. 目錄地圖（快速判讀）

- `Controls/`：可直接重用的 WPF 控制項（例如 PLC 狀態、登入、權限按鈕、列印頭相關面板）
- `Helpers/`：全域上下文與核心基礎設施（`SecurityContext`、`ComplianceContext`、`SqliteLogger`、`PlcContext` 等）
- `Services/`：帳號、驗證、主題等服務層實作
- `Models/`：權限、事件、資料模型與列舉
- `Themes/`：主題資源字典、色彩 Token、控制項樣式
- `Shell/`：Shell 導航與 metadata 契約（供多應用共用）
- `Docs/`：設計標準、重構進度、視覺 Token 收斂規範
- `Examples/`：主題系統示例

## 4. 核心設計與執行流

### 4.1 安全與權限

- `SecurityContext` 負責登入狀態、存取判斷與自動登出策略。
- 控制項層可透過 `SecuredButton` + `SecurityContext.CheckAccess(...)` 做雙層防護。
- 權限等級採 `AccessLevel`（Guest -> SuperAdmin）分層。

### 4.2 合規與日誌

- `ComplianceContext` 提供統一 API（`LogAuditTrail`、`LogOperation`、`LogEvent`、`LogPeriodicData` 等）。
- `SqliteLogger` 採佇列/批次寫入，兼顧效能與落盤可靠性。
- UI 即時日志透過 `LiveLogs` 提供資料繫結來源。
- 關機前必須 `ComplianceContext.Shutdown()`，確保待寫入資料完成落盤。

### 4.3 主題與視覺一致性

- 主題入口是 `Themes/Theme.xaml`。
- 新增/調整控制項時，優先使用語意 Token（`Surface.*`, `Text.*`, `Action.*`）。
- 設計方向明確要求：避免在控制項 XAML 中持續累積硬編碼色碼，Dark/Light 字典需同步。

### 4.4 Shell 複用方向

- `ShellNavigationService` + `ShellNavigationTargets` + `ShellRouteCatalog` 提供共通導航核心。
- 新 App 優先採 metadata + adapter 方式掛接，避免把 App 特定邏輯回灌到 Core。

## 5. 專案目前方向（Roadmap 導向）

從 `Docs/` 內容可看出目前主軸：

1. 控制項重構與生命週期穩定化（例如 Login、SecuredButton、LiveLogViewer）。
2. 視覺 Token 收斂，降低硬編碼色碼，強化暗/亮主題一致性。
3. 強化 Shell 化能力，讓第二個/後續 App 可用最少程式碼接入。

一句話方向：**把 Core 打造成「可長期演進、可跨專案重用、可追溯合規」的 UI 基座。**

## 6. 新人或 AI 介入時建議順序

1. 先讀 `README.md` 與 `Docs/CORE_UI_DESIGN_STANDARD.md`。
2. 讀 `Helpers/SecurityContext.cs`、`Helpers/ComplianceContext.cs` 理解安全與日誌核心。
3. 讀 `Themes/Theme.xaml`、`Themes/Colors.xaml`、`Themes/LightColors.xaml` 理解 Token 系統。
4. 若是新 App 整合，讀 `Shell/SECOND_APP_QUICKSTART.md`。
5. 修改控制項前，先比對 `Docs/CONTROLS_VISUAL_TOKEN_CONVERGENCE.md` 的收斂規則。

## 7. 開發守則（避免踩雷）

- 不要在 `Controls/*.xaml` 新增無語意的硬編碼色碼。
- 不要繞過 `ComplianceContext` 直接散落寫日誌。
- 不要把 App 專屬流程耦合進 Core（應放在 adapter/上層專案）。
- 任何涉及關閉流程的改動，需確認 `Shutdown()/Flush` 路徑仍完整。
- 若編譯失敗，先檢查外部 project references 與 `FeiyangWrapper` 實體檔案是否存在。

## 8. 30 秒摘要（給交接/AI）

這個目錄是一個 .NET 8 WPF 的工業 UI 核心庫，專長在 PLC 互動、權限、合規日誌、主題 Token 與 Shell 導航共用。它不是最終 App，而是上層應用的基礎平台。後續演進重點是「控制項穩定化 + 視覺 Token 收斂 + 多 App 複用能力」，所有改動都應優先維持可重用性、可追溯性與主題一致性。
