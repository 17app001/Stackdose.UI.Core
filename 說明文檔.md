# Stackdose.UI.Core 說明文檔

## 專案定位

`Stackdose.UI.Core` 是一個以 WPF 為基礎的工業 UI 核心元件庫，主要涵蓋：

- PLC 相關控制項與上下文管理
- 使用者登入、權限與自動登出
- FDA 21 CFR Part 11 導向的稽核與操作紀錄
- Dark/Light 主題資源與 UI 樣式

目標框架與型態可見於 `Stackdose.UI.Core/Stackdose.UI.Core.csproj`：

- `TargetFramework`: `net8.0-windows`
- `UseWPF`: `true`
- `PlatformTarget`: `x64`

## 目錄與模組

核心結構位於 `Stackdose.UI.Core/`：

- `Controls/`: UI 控制項（如 `PlcStatus`、`PlcLabel`、`LoginDialog`、`SecuredButton`、`SensorViewer`）
- `Helpers/`: 共享上下文與系統核心（`SecurityContext`、`ComplianceContext`、`SqliteLogger`、`PlcContext` 等）
- `Services/`: 業務服務（`UserManagementService`、`AdAuthenticationService`、`WindowsAccountService`）
- `Models/`: 權限、使用者、記錄模型與主題模型（`AccessLevel`、`UserAccount`、`LogEntry` 等）
- `Themes/`: 主題與控制項樣式（`Theme.xaml`、`Colors.xaml`、`LightColors.xaml`、`Themes/Components/*`）

## 重要核心流程

### 1) 認證與權限

認證核心在 `Stackdose.UI.Core/Helpers/SecurityContext.cs`：

- 提供 `Login` / `Logout` / `QuickLogin`
- 維護 `CurrentSession`
- 透過事件推送 UI 狀態變化：`LoginSuccess`、`LogoutOccurred`、`AccessLevelChanged`
- 支援自動登出（`EnableAutoLogout` + `AutoLogoutMinutes`）

使用者管理在 `Stackdose.UI.Core/Services/UserManagementService.cs`：

- 初始化 `Users`、`UserAuditLogs` 資料表
- 確保預設管理者帳號存在
- 提供建立、更新、停用、啟用、重設密碼與權限檢查

### 2) 合規與日誌

合規入口為 `Stackdose.UI.Core/Helpers/ComplianceContext.cs`：

- 統一記錄 `Audit`、`System`、`Operation`、`Event`、`PeriodicData`
- 提供可綁定的 `LiveLogs`（UI 即時顯示）
- 透過 `SqliteLogger` 實作批次寫入與定時刷新

SQLite 批次引擎在 `Stackdose.UI.Core/Helpers/SqliteLogger.cs`：

- 建立 `DataLogs`、`AuditTrails`、`OperationLogs`、`EventLogs`、`PeriodicDataLogs`
- 使用併發佇列 + Timer 定時 flush
- 提供統計與手動 flush/shutdown API

### 3) PLC 上下文

`Stackdose.UI.Core/Helpers/PlcContext.cs` 提供兩種路徑：

- `GlobalStatus`：全域 PLC 狀態
- Attached Property `Status`：容器層級繼承式注入

這讓控制項可在不同畫面結構中共用同一 PLC 連線來源。

### 4) 主題資源

主題入口為 `Stackdose.UI.Core/Themes/Theme.xaml`：

- 預設合併 `Colors.xaml`（Dark）
- 合併 `Components/Controls.Default.xaml` 與 `Components/Controls.Base.xaml`
- Light 主題採 `LightColors.xaml` 進行切換

## 與上層 App 的整合重點

1. 在 App 資源中合併 `Stackdose.UI.Core/Themes/Theme.xaml`。
2. 啟動階段初始化 `ComplianceContext`（觸發資料庫與 logger 初始化）。
3. 啟用並使用 `SecurityContext` 管理登入流程與 UI 權限狀態。
4. 關閉程式前呼叫 `ComplianceContext.Shutdown()` 以確保佇列資料落盤。
