# Stackdose.UI.Core — 方案說明

> 給接手的開發者或 AI 快速上手用。讀完這份文件約需 5 分鐘。

---

## 1. 這是什麼

**Stackdose.UI.Core** 是一套企業級 WPF 工業設備 UI 框架，目標是讓設備廠商能用最少程式碼快速建立符合 **FDA 21 CFR Part 11 合規稽核**要求的操作介面。

這不是單一應用，而是一個**框架產品**：
- `Core` / `Templates` 是基礎庫
- `DeviceFramework` 是組裝框架
- `App.*` 是範例/Demo
- `Tools.*` 是開發工具

## 4. 核心架構

### 4.1 Static Context Manager（不用 DI 容器）

框架採靜態 Context 管理全域狀態：

| Context | 職責 |
|---|---|
| `PlcContext` | 全域 PLC 實例、附加屬性傳遞 |
| `SecurityContext` | 登入/登出/權限/自動登出計時器 |
| `ComplianceContext` | 統一日誌 API（稽核、操作、事件、週期數據） |
| `SensorContext` | 感測器配置載入、警報狀態追蹤 |
| `ThemeManager` | 動態主題切換、WeakReference 控制項登錄 |

### 4.2 JSON 驅動設定（零程式碼上線路徑）

```
Config/
├── app-meta.json          ← Header、導航、Hot Reload 策略
└── Machine*.config.json   ← 每台設備：PLC IP/Port、Tags、Commands、Alarm/Sensor
```

`RuntimeHost` 讀取 JSON → `DeviceContextMapper` 轉換 → `DynamicDevicePage` 自動呈現 Labels/Commands。

### 4.3 資料流

```
PLC 輪詢: PlcStatus → ScanUpdated → PlcLabel 訂閱更新

命令執行: SecuredButton（權限檢查）→ ProcessCommandService → PLC Write

日誌:     ComplianceContext → SqliteLogger（非同步批次，5 秒 flush）→ LiveLogViewer
```

### 4.4 Shell 導航

```
ShellRouteCatalog → Overview / Detail / Log / User / Settings
NavigationOrchestrator 統一頁面切換
IShellAppProfile 定義 App metadata
```

### 4.5 權限等級

`Guest < Operator < Instructor < Supervisor < Admin < SuperAdmin`

## 6. 建立新設備 App（3 步驟）

### 方式 A：CLI（推薦）

```powershell
# 從方案根目錄執行
.\scripts\init-shell-app.ps1 -AppName "Stackdose.App.YourMachine" -DestinationRoot .

# 單頁設計版本
.\scripts\init-shell-app.ps1 -AppName "Stackdose.App.YourPage" -DestinationRoot . -SinglePageDesignerLocalEditable -DesignerLayoutPreset ThreeColumn
```

可用 DesignerLayoutPreset：`ThreeColumn`、`TwoColumn64`、`TwoByTwo`、`Blank`

### 方式 B：GUI 工具

執行 `Stackdose.Tools.ProjectGeneratorUI` — WPF 介面，可設定顏色、圖形、路徑後一鍵產生。

### 步驟
1. 產生新專案（CLI 或 GUI）
2. 修改 `Config/app-meta.json` 與 `Config/Machine*.config.json`
3. `dotnet build` 並執行

## 7. 關鍵守則

- **不要**在 `Controls/*.xaml` 寫硬編碼色碼，使用語意 Token
- **不要**繞過 `ComplianceContext` 直接散落寫日誌
- **不要**把 App 專屬邏輯耦合進 `UI.Core`
- 關閉 App 前必須呼叫 `ComplianceContext.Shutdown()` 確保日誌落盤
- 編譯失敗先確認 `Stackdose.Platform/` 各專案 reference 與 `FeiyangWrapper.dll` 是否存在

---

## 8. 擴充優先順序（DeviceFramework）

1. 純 JSON + `DynamicDevicePage`（最快上線）
2. 覆寫 `IRuntimeMappingAdapter` / `DefaultRuntimeMappingAdapter`
3. 覆寫 `RuntimeMapper.CreateDeviceContext(...)`
4. 用 `AppController.ConfigurePageFactory(...)` 換自訂 Device Page
5. 設定 `AppController.SettingsPage` 做客製 Settings

## 10. 30 秒摘要

這是一個 .NET 8 WPF 工業設備 UI **框架產品**。Core 提供控制項基礎庫，DeviceFramework 用 JSON 驅動快速組出設備 App，Tools 提供一鍵建立新專案的工具。開發者通常只需修改 JSON 設定即可上線，進階需求才覆寫 Adapter 或客製頁面。合規日誌（FDA 21 CFR Part 11）與 PLC 通訊（Mitsubishi FX3U）是框架的核心差異化能力。
