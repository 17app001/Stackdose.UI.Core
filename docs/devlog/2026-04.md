**refactor(B1+B2)** 基類統一 + 事件能力收斂（commit `b0e424d`）
- 新增 `PlcValueChangedEventArgs` 至 `Helpers/`（移出 PlcLabel，改為共用）
- 升級 `PlcControlBase`：加 `ValueChanged` event bus、`RaiseValueChanged()`、`CurrentValue` 屬性、`OnPlcConnected` / `OnPlcDataUpdated` 抽象方法、`OnGlobalStatusChanged` 熱更新
- 升級 `PlcEventContext`：加 `ControlValueChanged` static event + `PublishControlValueChanged()`（供未來 B5 Behavior Engine 訂閱）
- 5 個控件完成遷移（PlcLabel / PlcText / PlcStatusIndicator / AlarmViewer / SensorViewer）
  - 全部改用 `OnPlcControlLoaded` / `OnPlcControlUnloaded` / `OnPlcConnected` / `OnPlcDataUpdated` 覆寫
  - 移除各控件手動訂閱 PlcStatus 的樣板程式碼（約 -150 行）
  - 修正 PlcLabel.OnThemeChanged 遺漏 `override`（CS0114）
  - 移除 PlcText 與基類重複的 `SafeInvoke`（CS0108）
- XAML：5 個控件根元素從 `<UserControl>` 換成 `<base:PlcControlBase>`
- 編譯：Stackdose.UI.Core **0 errors、0 warnings**；所有相依專案全過

**refactor** 啟動「底層基礎 + Behavior Engine」重構（B0–B8）
- 新分支：`refactor/foundation-and-behavior`
- 觸發背景：Dashboard 模式完成後發現 PlcLabel 等控件無法表達「值變化時的反應行為」；追溯到底層 4 個結構問題（無共用基類、無統一控件事件出口、Shell 無策略化、文件漂移）
- 最終主旨：「讓不懂 XAML 的工程師或設計師，也能為 PLC 工業機台做出符合 FDA 21 CFR Part 11 稽核要求的監控介面」
- 方法：底層優先（B0–B3）→ Behavior 層（B4–B5）→ Designer 層（B6–B7）→ 文件回灌（B8）

**docs** 重構 meta 文件建立
- 新增 `docs/refactor/` 資料夾：README / PLAN / PROGRESS / HANDOFF 四份
- 產出 `docs/refactor/B0-control-inventory.md`：24 Core + 17 Templates 控件實況盤點
- 產出 `docs/refactor/B0-findings.md`：文件 vs 實況差異 20 條
